# Supplementary Reference Style Guide

Use this guide only as secondary style support. The master SAU/ICAR prompt remains the controlling instruction. Do not copy wording, data, references, treatment names, or conclusions from any reference document unless the user provides that same content as source material for the current table.

## Reference Documents Reviewed

- A compact agricultural Result/Discussion PDF containing short result blocks followed by literature-backed discussion blocks.
- A thesis chapter on Results and Discussion with numbered sub-sections, year-wise results, pooled results, correlation results, screening results, and economics.

## Observed Thesis-Style Structure

1. Begin broad chapters or major sub-sections with the experiment context before moving into individual tables.
2. Use numbered headings for complex thesis chapters, such as main experiment, parameter, first year, second year, pooled over periods and years, and economics.
3. For multi-year or multi-period tables, write first-year results first, second-year results next, and pooled-over-year or pooled-over-period results last.
4. For pest seasonal incidence, describe first appearance, gradual increase, peak period, decline, and crop stage or SMW/month where provided.
5. For screening and varietal reaction, compare hybrids/entries by damage or population level and classify only when the table gives enough basis.
6. For correlation tables, state direction and significance for each weather parameter and avoid causal explanation unless discussion is requested.
7. For economics, report gross realization, net realization, ICBR, and B:C ratio in descending practical importance when available.

## Result Family-Specific Writing Guardrails

1. Bioefficacy / management: use treatment-efficacy logic only for pesticide, acaricide, insecticide, fungicide, bioagent, spray, DAS/DAT, or pest/damage reduction tables. Write pre-treatment homogeneity if present, then spray-wise or pooled efficacy, lowest pest/damage treatment, supported at-par treatments, next best treatments, least effective treatment, and untreated check at the end.
2. Crop loss / yield loss / avoidable loss: use protected vs unprotected or treated vs untreated comparison. Report pest/damage level, yield in protected and unprotected plots, yield increase, avoidable loss, and per cent yield/crop loss. Do not use "best treatment" unless multiple treatments are actually compared.
3. Seasonal incidence / population dynamics: write first appearance, gradual increase, peak period with value, decline, and disappearance or crop stage. Do not use control or treatment-ranking wording.
4. Screening / varietal reaction / genotype evaluation: write variation among entries, least infested or resistant entries, moderately resistant/tolerant entries, susceptible entries, and reaction categories only when supplied or clearly supported.
5. Economics: emphasize yield, gross realization, net realization, cost of treatment, ICBR, and B:C ratio. The practical economic treatment may differ from the lowest pest-population treatment.
6. Correlation / regression: report positive or negative correlation, significance level, strongest relations, non-significant parameters, and regression equation/R2 only when supplied. Do not add biological cause in Results.
7. Survey / occurrence / distribution: report survey area, host/crop-wise occurrence, district/taluka/village-wise incidence, dominant pest/species, and notable occurrence only if supplied. Do not rank treatments.
8. Biology / life table / mass multiplication: report stage-wise duration, shortest or longest period according to parameter, fecundity, survival, multiplication, and host/sex/treatment comparison. Do not use pesticide-efficacy wording.
9. Bioassay / toxicity / resistance: write toxicity order, lowest LC50 as highest toxicity, highest LC50 as lowest toxicity, resistance ratio, fiducial limits, and slope only when relevant. Do not use field spray-wise style.
10. Natural enemy / biological control: write predator, parasitoid, predator:prey ratio, release, predation, parasitization, suppression, or multiplication logic. Use biological control terms rather than chemical bioefficacy terms unless the table is pesticide compatibility/toxicity.
11. Phytotoxicity / safety / compatibility: write safety parameter, lowest phytotoxicity or safest treatment, harmful treatment if any, and dose-wise trend. Do not call a treatment best for pest control unless pest-control data are supplied in that table.

## Observed Result-Writing Habits

1. Start with a table callout: "The data are presented in Table..." or "The data presented in Table... revealed..."
2. Use the parameter and pest name repeatedly instead of vague pronouns.
3. Report exact original values with units beside the treatment or period being interpreted.
4. Keep transformed values out of the narrative. Use transformed values only to understand significance, CD/SEm, ANOVA status, or grouping.
5. Use "pooled over periods", "pooled over years", or "pooled over periods and years" when that structure is present.
6. Put untreated control/check at the end of treatment comparison paragraphs.
7. For before-spray observations, mention homogeneity or non-significance before post-spray efficacy.
8. Use "more or less similar trend" only when the table genuinely shows a repeated pattern across sprays, years, or periods.
9. Place values immediately beside the treatment or phrase being ranked, usually as `Treatment name (original value unit)`.
10. Follow a fixed ranking order: best treatment, at par treatments, next best or middle-tier treatments, comparatively less effective treatments, and worst treatment/check.

## User Style Update

1. The result write-up must mention only original values. Transformed values are allowed only for deciding statistical significance and should not appear in the final paragraph.
2. If values are written as `original (transformed)`, the write-up must use only the original value outside parentheses.
3. If the transformed value carries the statistical letter, that letter can be used only to decide "at par" or "differed significantly"; neither the transformed value nor the letter should appear in the narrative.
4. Interaction information should not be written earlier in the paragraph. It must be written only after the particular table, year, period, spray, or pooled data has been fully interpreted.
5. A typical paragraph should average about 20-30 words per sentence, with approximately 82 per cent passive voice and only 18 per cent active voice.
6. Use repeated thesis phrases such as "The data presented in Table...", "which was statistically at par with...", "significantly lowest", "significantly maximum", and "pooled over periods and years".
7. Use the following transition words where suitable: "however", "on the other hand", "conversely", and "moreover".
8. Paragraphs should usually contain 4-6 sentences and focus on one year, one spray, one period, or one pooled dataset at a time.
9. Damage, pest population, yield, and economics should remain in separate result blocks unless the table itself combines them.
10. Discussion-style literature comparison should be avoided unless the user specifically asks for discussion.

## Required Interaction Placement

Use this order when interaction is present:

1. Table callout and significance statement for the main parameter.
2. Best treatment/period/hybrid with original value.
3. Statistically at par treatments interpreted from grouping letters, without printing those letters.
4. Next best, moderate, mediocre, or comparatively less effective treatments.
5. Worst treatment, susceptible entry, maximum damage, or untreated check.
6. Pooled data interpretation, if available.
7. Interaction sentence at the end.

Preferred interaction sentence pattern:

"The interaction between [factor A] and [factor B] ([Factor A] x [Factor B]) was found [significant/non-significant] with respect to [parameter]."

For treatment x year interaction:

"The interaction between treatment and year (Treatment x Year) was found non-significant with respect to [parameter]. This indicated that treatment performance remained consistent over the years."

Do not place either sentence before the main table findings. Do not narrate the writing process; state only the statistical fact directly.

## Middle-Tier Performance Dictionary

Use middle-tier words only after the best and next-best treatments have been reported:

1. "Mediocre" for treatments or periods falling clearly in the middle rank.
2. "Moderate" or "moderately effective" for acceptable but not superior control or incidence.
3. "Intermediate" for statistically or numerically middle action.
4. "Appreciable" for noticeable but lower-than-best control.
5. "Satisfactory" for acceptable control below the top group.
6. "Comparatively less effective" for lower-middle treatments.
7. "Relatively inferior" or "relatively poor" for treatments below the middle group but not necessarily equal to untreated control.
8. "Next best" or "next better" for second-ranked treatments or periods.

## Observed Discussion Placement

1. Discussion should follow the result, not interrupt the statistical interpretation.
2. Use discussion only when the user requests it or provides literature/source references.
3. Discussion paragraphs compare present findings with earlier workers using phrases such as "in agreement with", "in conformity with", "supported by", "comparable with", or "partially in agreement with".
4. If no comparable reference is provided, state that the result could not be discussed with literature instead of inventing citations.

## Guardrails From References

1. Preserve treatment names, crop names, pest names, years, locations, SMW, DAS, DOS, WAS, units, and table numbers exactly as supplied.
2. Do not merge result and discussion in the same paragraph unless the user explicitly asks for a combined section.
3. Do not add biological explanations or literature names that are absent from the user-provided source.
4. Avoid correcting grammar inside treatment names, quoted table headings, cultivar names, accession numbers, or chemical formulations.
5. When scanned or legacy files are unclear, explicitly mention unreadable values rather than estimating.
6. Never invent interaction significance. Write interaction only when the table provides that information.
7. Never use transformed values as the displayed biological result.
8. Never reproduce DNMRT, DMRT, or grouping letters such as `a`, `ab`, `bc`, or `c` in the final result write-up.
9. Do not write the term "DNMRT" in the narrative unless the user explicitly asks for the statistical method name.
